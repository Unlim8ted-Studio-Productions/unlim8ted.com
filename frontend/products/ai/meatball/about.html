About the Meatball AI

The current staged runtime

Meatball is not one generator answering everything. The page runs a sequence of small models in the browser, keeps short conversational memory, rewrites followups when needed, routes math away from the normal generator, applies answer overrides, and drives the emotion renderer separately from the QA system.

Current Message Pipeline
This is the architecture the page is actually using right now, including runtime fallbacks and post-processing.
Input
Raw user message enters the runtime
Example: what is the glich, what is it, what is 1+1, cats vs dogs.
↓

Reaction classifier

Predicts Meatball's response state: neutral, excited, confused, suspicious, angry, sad, or overwhelmed.

Complexity classifier

Predicts route shape: normal_qa, list, compare, multi_part, followup, smalltalk, unknown.

Math classifier

Predicts general vs math. The runtime also has a heuristic backup for obvious arithmetic.

Subject finder

Extracts the current subject span from the current message only, not the whole conversation blob.

↓

Subject inserter

Sees the raw question, current subject, complexity label, previous subject, and last answer. It decides whether to keep the input or rewrite it using the remembered subject. 

question: does it have clothes
previous subject: The Glitch

rewritten:
does The Glitch have clothes

Runtime memory

The page keeps short JS memory: history, recent subjects, lastAnswer, previousReaction, angryStreak, and sauceAttackCooldown. 

{
  history: last 8 messages,
  subjects: last 5 non-NONE subjects,
  lastAnswer: "...",
  previousReaction: "neutral",
  angryStreak: 0,
  sauceAttackCooldown: 0
}

Answer branch
Neutral generator
The normal generator answers rewritten questions, list prompts, followups, multi-part splits, and many smalltalk-style replies.
Answer branch
Math system
Obvious arithmetic can be handled directly with safe evaluation. Otherwise the ONNX math translator is used and then safely evaluated.

Compare handler
Compare prompts can parse two subjects and currently answer by generating a short description for each side and appending both.

Post-processing
The raw answer gets cleaned and overridden
Punctuation cleanup, capitalization fixes, line-preserving list formatting, and hard response overrides all happen after routing.

↓
Animation
Emotion and animation are separate from QA
The reaction model controls the renderer state. Angry escalation can trigger a dedicated cutscene path. Error paths can trigger the glitch animation. The text answer does not directly decide the route.
↓
UI output
Final answer + final emotion + updated memory
After the response is shown, the page stores the last answer, remembers any detected subject, updates angry cooldown logic, and keeps only a short sliding window of history.
Current Special Behaviors
These are runtime rules that exist in addition to the learned models.
Math fallback: obvious expressions like what is 1+1 can route to math even if the classifier misses.
List behavior: list/facts/examples/features requests reshape the question before generation, then render each sentence as bullets.
Compare behavior: if two subjects are parsed, the runtime asks what is X and what is Y and appends both outputs.
Followup rewrite: pronoun followups like what is it or does it have clothes can be rewritten using the previous subject.
Micro overrides: ok and okay reply with Yep. and yep replies with Yes.
Angry escalation: repeated angry reactions can bypass normal QA and trigger the sauce attack cutscene with cooldown protection.
